Microservice Documentation

API Endpoints

User Service

This service handles user-related operations like login, registration, and user profile management.

  • GET /users - Retrieve a list of all users
  • GET /users/{id} - Get user details by ID
  • POST /users - Create a new user
  • PUT /users/{id} - Update user details
  • DELETE /users/{id} - Delete a user

Item Service

This service manages items such as adding, retrieving, and deleting items.

  • GET /items - Retrieve a list of items
  • GET /items/{id} - Get item details by ID
  • POST /items - Add a new item
  • PUT /items/{id} - Update item details
  • DELETE /items/{id} - Delete an item

How to Use

The microservices are deployed and available via the following URLs:

  • User Service: http://localhost/users
  • Item Service: http://localhost/items

For each service, the respective API endpoints are available as mentioned above.

System Architecture

This microservice architecture consists of a user service and an item service, which are managed through Nginx as a reverse proxy. The Nginx configuration handles routing requests to the respective services.
